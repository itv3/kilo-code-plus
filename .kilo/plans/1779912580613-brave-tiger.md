# Plan: JetBrains Session Scroll Question Overlay

## Goal
When the JetBrains session transcript is scrolled away from the bottom and the active session is awaiting user input for a normal question, replace the floating scroll-to-bottom icon with an orange question icon. The overlay should still jump to the bottom when clicked.

## Current Findings
- `SessionScroll` (`packages/kilo-jetbrains/frontend/src/main/kotlin/ai/kilocode/client/session/scroll/SessionScroll.kt`) owns the floating overlay as `jump: JBLabel`, currently using `ScrollButtonIcon.create()` and showing only when `component.viewport.view === messages && !atBottom()`.
- `SessionUi` (`.../session/SessionUi.kt`) already maps `SessionState.AwaitingQuestion` to `SessionActivityKind.QUESTION` or `SessionActivityKind.PLAN` via `activityKind()`.
- Question state changes flow through `SessionModelEvent.StateChanged -> onStateChanged(state)`, where `scroll.refresh()` already runs.
- Existing tests in `SessionScrollTest` cover that the scroll overlay appears while a user is in the middle during questions, but they only assert visibility, not which icon is shown.
- Existing history badges use `SessionActivityKind.QUESTION` and `UiStyle.Colors.activityBadgeBg()/activityBadgeFg()`, but the requested overlay specifically asks for an orange question icon, so this should use a warning/orange semantic color rather than the blue activity badge color.

## Implementation Steps
1. Extend `SessionScroll` with lightweight state for whether the overlay should show a question indicator.
   - Add an EDT method such as `setQuestionPending(value: Boolean)`.
   - Store the current value in a private `question` boolean.
   - Update the label icon and tooltip only when the mode changes.
   - Keep click behavior unchanged: any overlay click calls `jumpBottom()`.

2. Add a question icon variant for the overlay.
   - Prefer reusing the existing `scroll-bottom.svg` shape only for scroll mode and create a new matching 40x40 SVG for question mode, e.g. `/icons/scroll-question.svg` and `_dark.svg` if needed.
   - Use explicit SVG colors as required by JetBrains icon loading rules.
   - Use the existing `colorizeIfPossible` path where possible so runtime foreground/background can come from theme APIs.
   - For the orange background, use `UiStyle.Colors.warningLabelForeground()` or a named Kilo color token with a warning/orange fallback if a better platform semantic color is not available.
   - Foreground should remain high contrast, likely white.

3. Wire `SessionUi.onStateChanged(state)` to update the scroll overlay mode.
   - Set question mode only for `SessionState.AwaitingQuestion` where the question is not a plan follow-up.
   - Use the existing `planFollowup()` helper so plan follow-ups continue using the normal scroll-down overlay.
   - Call the scroll setter before or around `scroll.show(body(state))`/`refresh()` so state transitions immediately update the overlay.

4. Preserve visibility semantics.
   - `SessionScroll.updateJump()` should still decide visibility from transcript/body and bottom position.
   - The icon mode should not make the overlay visible when the transcript is at bottom or outside the message body.
   - When the question is replied/rejected and state returns to idle/busy/etc., the next state change should switch the icon back to scroll-down mode while preserving current scroll position.

5. Add tests in `SessionScrollTest`.
   - Assert that during a normal `QuestionAsked` while user is in the middle, `jumpButton().isVisible` remains true and its icon changes from the original scroll icon to the question icon.
   - Assert that clicking the question overlay still scrolls to bottom and hides the overlay.
   - Assert that replying to the question while still away from bottom switches the overlay back to the scroll icon if it remains visible.
   - Assert that plan follow-up questions keep the normal scroll icon, if constructing a plan follow-up is straightforward with existing helpers.

6. Add/update localized strings only if a separate tooltip is introduced.
   - Preferred minimal behavior: keep `session.scroll.bottom=Scroll to bottom` as the tooltip because the action is unchanged.
   - If a new tooltip is desired, add `session.scroll.question` to `KiloBundle.properties` and the translation files can fall back only if the project has an established fallback pattern; otherwise avoid adding a new string.

7. Add a changeset if no existing JetBrains scroll changeset should cover this.
   - Package is private but existing changesets include `@kilocode/kilo-jetbrains`, so add a patch changeset like: `Show a question indicator on the JetBrains session scroll overlay when user input is needed.`

## Verification
- Run targeted JetBrains tests: `./gradlew test --tests ai.kilocode.client.session.SessionScrollTest` from `packages/kilo-jetbrains/` if Gradle supports the class filter in this project.
- If the targeted test command is unavailable or too narrow, run `bun run test` from `packages/kilo-jetbrains/`.
- Run `bun run typecheck` from `packages/kilo-jetbrains/` for Kotlin compilation.

## Risks / Notes
- Do not change backend, RPC, or session model state; all required state already exists in the frontend UI.
- Keep all Swing access on EDT, following the existing `SessionScroll`/`SessionUi` pattern.
- Avoid hardcoded runtime Swing colors; hardcoded hex is acceptable only inside SVG assets per the plugin icon guidelines.
