# Plan: `kilodev` launcher + `kilo dev-setup` / `kilo dev-alias` helpers

> **Status: shipped** — this document now describes the implementation that
> landed on `feat/cli-local-run` in commits `59d54806f6c`
> (`feat(cli): add kilodev launcher + dev-alias helper`) and `dd7475f6827`
> (`feat(cli): add interactive kilodev dev-setup command`). It is kept as a
> design record. See the actual files in `bin/` and
> `packages/opencode/src/kilocode/cli/dev-setup.ts`.

Goal: make it trivial for contributors to run the in-repo CLI
(`bun run --cwd packages/opencode … src/index.ts`) from any folder, with a
single-line shell config, on macOS/Linux/Windows.

Before this change, users hand-wrote a zshrc function like:

```zsh
kilodev() {
  PWD="$PWD" bun run --cwd /Users/alex/.../packages/opencode --conditions=browser src/index.ts --project "$PWD" "$@"
}
```

We replaced that with a committed launcher plus two CLI helpers:
`dev-setup` (interactive, idempotent rc-file installer) and `dev-alias`
(scriptable one-liner emitter).

## Shipped artifacts

1. Cross-platform launcher under `bin/`:
   - `bin/kilodev` (bash, executable) for macOS/Linux/WSL/Git Bash
   - `bin/kilodev.cmd` (batch) for Windows cmd.exe / PowerShell / Windows Terminal
2. `kilo dev-setup` — interactive installer that writes a marker-delimited
   alias block into the user's rc file, so repeat runs update rather than
   duplicate.
3. `kilo dev-alias [shell]` — hidden subcommand that just prints the
   shell-specific one-liner (used by scripts and the `dev-setup` preview).
4. Documentation in `CONTRIBUTING.md` under "Developing Kilo CLI".

## Design notes (verified bun behavior)

- `bun --cwd=X` changes `process.cwd()` to X but **preserves the inherited
  `PWD` env var**. So the legacy zshrc's `PWD="$PWD"` prefix is redundant
  and we drop it.
- Kilo's TUI resolves the project dir from `args.project`; if absent, it
  uses `process.cwd()` (= `packages/opencode` when we pass `--cwd` there).
  We therefore inject `--project "$PWD"` **only when launching the TUI** —
  yargs is strict and only the default/TUI and `stats` commands declare
  `--project`, so unconditionally passing it would break `kilodev run …`,
  `kilodev serve`, `kilodev dev-alias …`, etc.
- `Installation.isLocal()` is already true when running via
  `bun run src/index.ts`, so `dev-setup` / `dev-alias` gate on it.

## Files

### `bin/kilodev` (new, executable, LF)

```bash
#!/usr/bin/env bash
set -e
dir="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
export KILO_DEV_REPO="$dir"
# Find the first non-flag arg. If there isn't one, the user is launching the
# TUI, so point it at the caller's directory via the project positional.
# Otherwise forward args untouched so subcommands (run, dev-alias, …) don't
# collide with --project (only TUI/stats accept --project under yargs strict).
first=
for a in "$@"; do
  case "$a" in
    --) break ;;
    -*) ;;
    *) first="$a"; break ;;
  esac
done
if [[ -z "$first" ]]; then
  exec bun run --cwd "$dir/packages/opencode" --conditions=browser src/index.ts --project "$PWD" "$@"
fi
exec bun run --cwd "$dir/packages/opencode" --conditions=browser src/index.ts "$@"
```

- Self-locating via `BASH_SOURCE` so symlinks, aliases, and direct
  invocation all work.
- `KILO_DEV_REPO` env var lets `dev-setup` / `dev-alias` discover the repo
  root without extra probing.
- Conditional `--project` injection avoids yargs "Unknown argument"
  failures on non-TUI subcommands.
- `exec` replaces the shell with bun so signals (Ctrl-C) and exit codes
  propagate cleanly.

### `bin/kilodev.cmd` (new, LF — cmd.exe tolerates LF)

```bat
@echo off
setlocal enabledelayedexpansion
set "dir=%~dp0.."
set "KILO_DEV_REPO=%dir%"
rem Find the first non-flag arg. If there isn't one, the user is launching the
rem TUI, so point it at the caller's directory via --project. Otherwise
rem forward untouched so subcommands don't collide with --project.
set "first="
for %%a in (%*) do (
  if not defined first (
    set "arg=%%~a"
    if "!arg:~0,1!" neq "-" set "first=%%~a"
  )
)
if not defined first (
  bun run --cwd "%dir%\packages\opencode" --conditions=browser src/index.ts --project "%CD%" %*
) else (
  bun run --cwd "%dir%\packages\opencode" --conditions=browser src/index.ts %*
)
set "code=%ERRORLEVEL%"
endlocal & exit /b %code%
```

- `%~dp0` expands to the directory of the `.cmd` file (with trailing `\`),
  so `%~dp0..` is the repo root.
- `%CD%` is the caller's current directory — same role as `$PWD` on Unix.
- `enabledelayedexpansion` is required for the `!arg:~0,1!` flag test.
- Capturing `ERRORLEVEL` into `code` before `endlocal` preserves bun's
  exit code across the scope boundary.

### `packages/opencode/src/kilocode/cli/dev-setup.ts` (new)

Exports two yargs commands: `DevSetupCommand` (the primary interactive
installer) and `DevAliasCommand` (a hidden raw-line emitter kept for
scripting). The file lives under `packages/opencode/src/kilocode/`, so no
`kilocode_change` markers are needed per `AGENTS.md`.

Key internals:

```ts
type Shell = "zsh" | "bash" | "fish" | "powershell"

const MARKER_START = "# >>> kilodev launcher >>>"
const MARKER_END = "# <<< kilodev launcher <<<"

function aliasLine(shell: Shell, repo: string): string {
  const sh = path.join(repo, "bin", "kilodev")
  const bat = path.join(repo, "bin", "kilodev.cmd")
  const q = (s: string) => `'${s.replaceAll("'", `'\\''`)}'`
  if (shell === "fish") return `alias kilodev ${q(sh)}`
  if (shell === "powershell") return `function kilodev { & '${bat.replaceAll("'", "''")}' @args }`
  return `alias kilodev=${q(sh)}`
}

async function detectRepo(): Promise<string> {
  const hint = process.env.KILO_DEV_REPO
  if (hint) return hint
  const walk = async (dir: string): Promise<string> => {
    const candidate = path.join(dir, "packages", "opencode", "package.json")
    if (await Bun.file(candidate).exists()) return dir
    const parent = path.dirname(dir)
    if (parent === dir) throw new Error("cannot locate repo root")
    return walk(parent)
  }
  return walk(path.dirname(fileURLToPath(import.meta.url)))
}
```

Notes:

- `q()` properly shell-escapes the repo path: single quotes in Unix paths
  become `'\''`, and PowerShell's single-quote doubling (`''`) is applied
  separately. This prevents broken/unsafe aliases on paths like
  `/Users/O'Brien/code/kilocode`.
- Repo detection uses `await Bun.file(candidate).exists()` — not
  `.size > 0`, which would misbehave on an empty marker file and performs
  a blocking sync stat.
- `KILO_DEV_REPO` (set by the launcher) is the fast path; the walk-up is
  a fallback for when `dev-alias`/`dev-setup` is invoked without going
  through `bin/kilodev`.

`DevSetupCommand` writes the alias inside a marker block:

```
# >>> kilodev launcher >>>
alias kilodev='/path/to/kilocode/bin/kilodev'
# <<< kilodev launcher <<<
```

On re-run, `applyBlock` replaces the existing block instead of appending,
so repeated installs stay idempotent. A timestamped backup is written to
`<rc>.kilodev.bak.<YYYYMMDD-HHMMSS>` on update. Non-TTY invocations print
the snippet to stderr with instructions rather than prompting, so the
command is still pipe-friendly.

`DevAliasCommand` is the same `aliasLine()` result, printed to stdout.
`describe: false` hides it from `--help` since it's an implementation
detail for scripts and docs.

### `CONTRIBUTING.md`

New subsection under "Developing Kilo CLI" titled "Running Kilo CLI from
any folder":

1. Recommended path — interactive installer:

   ```bash
   # from the repo root, one time:
   ./bin/kilodev dev-setup
   ```

   Auto-detects `$SHELL`, previews the exact line, asks for confirmation,
   writes a marker-delimited block to the rc file, and backs up the prior
   version. Supports `--shell`, `--rc`, `--yes`, `--print`, `--dry-run`.

2. Scriptable alternative (raw snippet):

   ```bash
   ./bin/kilodev dev-alias zsh
   # => alias kilodev='/path/to/kilocode/bin/kilodev'
   ```

   Caller is responsible for idempotency — prefer `dev-setup` for
   interactive use to avoid duplicate lines on re-runs.

3. Manual alternatives (no CLI invocation):
   - Unix: `alias kilodev='/path/to/kilocode/bin/kilodev'` in `~/.zshrc`
     / `~/.bashrc`, or `fish_add_path /path/to/kilocode/bin`.
   - Windows: add `C:\path\to\kilocode\bin` to PATH, or
     `function kilodev { & "C:\path\to\kilocode\bin\kilodev.cmd" @args }`
     in `$PROFILE`.

4. Note that `kilodev <args>` is identical to `bun dev <args>` except it
   works from any directory and (for TUI launches only) passes
   `--project "$PWD"` so the TUI opens in the caller's folder.

## Alternatives considered

- **Bun-based `script/dev.ts` with shell wrappers**: adds a bun-in-bun
  indirection and an extra file; the small shell scripts are simpler and
  don't need a bun startup for the launcher itself.
- **`bin` entry in `@kilocode/cli/package.json`**: pollutes the published
  package with a dev-only command.
- **Embed in `kilo completion` output**: the completion command would
  need to know which repo checkout to target and hits a chicken-and-egg
  for users who don't yet have kilo installed. Keeping `dev-setup` and
  `dev-alias` as sibling commands is clearer.
- **Raw `dev-alias >> ~/.zshrc` as the only path**: simpler, but not
  idempotent — users who re-run get duplicate alias lines. We kept
  `dev-alias` for scripts/CI but added `dev-setup` with marker blocks as
  the recommended interactive UX.

## Style / conventions

- Single-word identifiers (`dir`, `sh`, `bat`, `rc`, `hint`, `line`,
  `candidate`, `parent`, `q`, `code`, `first`) per `AGENTS.md`.
- `const` only — no `let`.
- No try/catch in the new TS file.
- `kilocode_change` markers only in `src/index.ts` (the only shared file
  touched); not in `bin/*` or `src/kilocode/*`.
- LF line endings everywhere (`.gitattributes` already enforces
  `* text=auto eol=lf`).

## Validation (performed on the shipped change)

1. `cd /tmp && /path/to/kilocode/bin/kilodev --help` — prints kilo help,
   TUI does not launch, `--project` is not injected (no yargs error).
2. `cd /tmp && /path/to/kilocode/bin/kilodev` — opens TUI with project
   = `/tmp` via the injected `--project`.
3. `/path/to/kilocode/bin/kilodev dev-alias zsh` — prints
   `alias kilodev='.../bin/kilodev'` with proper escaping.
4. `/path/to/kilocode/bin/kilodev dev-alias powershell` — prints the
   PowerShell function with doubled single quotes.
5. `/path/to/kilocode/bin/kilodev dev-setup --dry-run` — shows the block
   it would write without touching the rc file.
6. `bun turbo typecheck` — passes.
7. `bun run check-kilocode-change` (from `packages/kilo-vscode/`) and
   `bun run script/check-opencode-annotations.ts` (from root) — the
   markers in `src/index.ts` pass both checks.
8. Windows smoke test: `bin\kilodev.cmd --help` from `C:\tmp` forwards
   cleanly; exit code preserved across `endlocal`.
