# Plan: Extract Suggestion Feature to Kilo-Specific Directories

## Goal

Move the "suggest code review" feature implementation out of core OpenCode paths into `packages/opencode/src/kilocode/suggestion/`, leaving only minimal hooks in shared files. This minimizes merge conflicts when syncing with upstream OpenCode.

- **10 modified core files** with ~206 lines added, ~22 lines modified
- **5 test files** (3 new, 2 modified)

## Strategy

1. **Move** all new files to `kilocode/suggestion/`
2. **Extract** substantial logic from high-conflict-risk shared files into `kilocode/suggestion/` helpers
3. **Leave** only thin hooks (imports + 1-5 line delegations) in core files
4. **Move** test files to `test/kilocode/suggestion/`

## Risk Notes

- **TUI reactivity:** SolidJS `createMemo` / `produce` / `reconcile` must receive correct store types when extracted. Test TUI after the sync handler extraction.
- **Import path aliases:** `@/kilocode/suggestion` must resolve correctly via the `@/*` → `./src/*` alias defined in `tsconfig.json`. This is the same pattern used by existing `@/kilocode/*` imports.
- **API path preserved:** Suggestion routes stay at `/suggestion` (not `/kilocode/suggestion`) to avoid SDK regeneration and client-side changes.
- **`remote-sender.ts`** is in `kilo-sessions/` which is Kilo-specific — no extraction needed, just update import paths.
